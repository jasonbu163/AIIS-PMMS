# 业务范围

## 核心业务目标

本项目围绕三个核心目标建设：

1. 每日管理生产过程中的剩余物料。
2. 每日生产前提供备料清单，辅助生产备料工序。
3. 通过每日和每月的用料、剩料、废料数据，为后续绩效计算提供依据。

## 第一阶段范围

第一阶段建议覆盖以下能力：

| 模块 | 说明 |
| --- | --- |
| 基础资料 | 物料、产品、产线、班组、员工、仓库、库位、废料原因。 |
| 标准用料 | 产品 BOM 或标准用料清单。 |
| 生产任务 | 每日生产任务、工单、计划数量、责任产线和班组。 |
| 备料清单 | 根据生产任务和标准用料生成每日备料需求。 |
| 领料记录 | 记录实际从仓库或线边仓领出的物料。 |
| 用料记录 | 记录生产过程实际消耗的物料。 |
| 剩料登记 | 记录生产结束后的剩余物料、可复用状态和处理方式。 |
| 废料登记 | 记录废料数量、原因、责任维度和备注。 |
| 退料记录 | 记录剩余物料退回仓库、线边仓或转入下次使用。 |
| 日结报表 | 汇总每日计划用量、实际用量、剩料、废料和差异。 |
| 月结报表 | 汇总月度用料、剩料、废料、损耗率和绩效基础数据。 |

## 暂不纳入第一阶段

以下能力可以作为后续阶段，不建议第一版就做：

- 完整采购管理。
- 完整财务成本核算。
- 复杂高级排产。
- 完整仓库波次、路径、上架策略。
- 自动化设备控制。
- 复杂绩效规则引擎。

## 核心单据

第一阶段建议优先设计这些单据：

- 生产工单
- 每日备料单
- 领料单
- 用料记录
- 剩料登记单
- 废料登记单
- 退料单
- 日结表
- 月结表
- 绩效数据汇总表

## 典型用户

| 角色 | 使用场景 |
| --- | --- |
| 生产主管 | 查看每日备料、用料、剩料、废料和班组差异。 |
| 备料人员 | 根据备料清单准备生产所需物料。 |
| 生产人员 | 登记实际用料、剩料和废料。 |
| 仓库人员 | 处理领料、退料、库存确认。 |
| 绩效核算人员 | 获取日/月统计数据，作为绩效计算输入。 |
| 管理人员 | 查看物料损耗、异常趋势和改进空间。 |

## 关键业务闭环

```text
计划用料
  ↓
备料
  ↓
领料
  ↓
生产消耗
  ↓
剩料确认
  ↓
废料确认
  ↓
退料 / 复用
  ↓
日结
  ↓
月结
  ↓
绩效数据
```
