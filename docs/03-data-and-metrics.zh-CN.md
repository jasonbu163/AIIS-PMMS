# 数据与指标

## 统计目标

系统需要把每日生产现场的物料变化记录成可核算的数据。核心关注点不是“有没有填表”，而是能否回答以下问题：

- 计划应该用多少物料？
- 实际领了多少物料？
- 实际生产消耗了多少物料？
- 生产后剩余多少物料？
- 产生了多少废料？
- 计划与实际之间差异多少？
- 差异属于哪个日期、工单、产品、产线、班组、人员或原因？

## 核心数据维度

| 维度 | 说明 |
| --- | --- |
| 日期 | 日结和月结的基础时间维度。 |
| 工单 | 追踪每一批生产任务的物料使用情况。 |
| 产品 | 分析不同产品的物料消耗和损耗。 |
| 物料 | 分析每种物料的计划、领用、消耗、剩余和废料。 |
| 产线 | 分析不同产线的物料利用情况。 |
| 班组 | 用于后续绩效计算和责任归属。 |
| 员工 | 如果绩效需要到个人，可记录操作人员。 |
| 仓库 / 库位 | 追踪领料、退料和剩料流向。 |
| 废料原因 | 分析废料来源和改善方向。 |

## 核心数量口径

| 指标 | 含义 |
| --- | --- |
| 计划用量 | 根据生产任务数量和 BOM / 标准用料计算出的理论需求。 |
| 备料数量 | 生产前实际准备的物料数量。 |
| 领料数量 | 从仓库或线边仓实际领出的数量。 |
| 实际用量 | 生产过程实际消耗的数量。 |
| 剩料数量 | 生产结束后未消耗、仍可确认的剩余数量。 |
| 退料数量 | 剩料退回仓库、线边仓或转入后续复用的数量。 |
| 废料数量 | 无法继续正常使用，需要报废或单独处理的数量。 |
| 差异数量 | 计划、领料、实际用料、剩料、废料之间的差额。 |

## 建议指标

第一阶段可以先沉淀以下指标：

| 指标 | 建议计算方式 |
| --- | --- |
| 用料差异 | 实际用量 - 计划用量 |
| 领用差异 | 领料数量 - 计划用量 |
| 剩料率 | 剩料数量 / 领料数量 |
| 废料率 | 废料数量 / 领料数量 |
| 物料利用率 | 实际用量 / 领料数量 |
| 综合损耗数量 | 废料数量 + 无法解释的差异数量 |
| 综合损耗率 | 综合损耗数量 / 领料数量 |

具体绩效公式可以后续再定，但第一阶段必须保证基础数据可追溯、可汇总、可复核。

## 日结输出

每日应能按日期、工单、产线、班组、产品、物料输出：

- 计划用量
- 备料数量
- 领料数量
- 实际用量
- 剩料数量
- 退料数量
- 废料数量
- 差异数量
- 废料原因
- 责任班组或责任人

## 月结输出

每月应能按月份、产线、班组、产品、物料输出：

- 月度计划用量
- 月度实际用量
- 月度剩料数量
- 月度废料数量
- 月度损耗数量
- 月度损耗率
- 班组或产线对比
- 异常物料排行
- 废料原因排行

## 数据质量要求

为了让后续绩效计算可信，第一阶段要重点保证：

- 每条用料、剩料、废料记录必须关联日期和工单。
- 每条记录必须关联物料。
- 涉及责任归属时，必须关联产线、班组或人员。
- 废料必须记录原因分类。
- 日结后数据原则上不应随意修改，应保留调整记录。
- 月结数据应来源于日结数据汇总，而不是手工重新填写。
