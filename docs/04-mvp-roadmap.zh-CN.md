# MVP 路线图

[English Roadmap](04-mvp-roadmap.md)

> 当前更新：本项目第一版已经收敛为 backend-first 的激光开料余料管理后端。具体实施跟踪以根目录 `PLAN.md` 为准；本文保留 MVP 阶段视角，并同步当前已确定的后端、数据库和交付边界。

## 目标

项目初始化完成后，下一阶段应把“激光开料余料管理系统”的业务设想收敛成可开发的 MVP。MVP 不追求完整 ERP、完整 MES 或完整 WMS，而是先完成激光开料现场的整料 / 余料 / 废料闭环和后端 API 契约。

## 阶段 0：项目初始化

当前阶段目标：

- 明确系统名称和英文简称。
- 明确系统不做什么。
- 沉淀业务范围、术语关系、数据指标和路线图。
- 建立 Git 仓库和基础 README。

完成标准：

- 根目录存在中英文 README。
- `docs/` 目录存在业务文档入口。
- 文档覆盖系统定位、术语关系、业务范围、数据指标和 MVP 路线。
- Git 仓库已初始化。

## 阶段 1：后端核心契约

目标是把业务语言变成可开发、可被 Qt / Vue 前端消费的后端契约。

建议产出：

- API 契约：统一响应信封、错误码、分页结构、OpenAPI 输出。
- 认证边界：JWT Bearer、refresh token、用户角色、受保护业务接口。
- 核心流程：备料、Template.xlsx 导出、生产报告导入、余料确认、废料确认、日结。
- 核心状态：`draft`、`generated`、`uploaded`、`parsed`、`confirmed`、`settled`、`voided`。
- 样例口径：字段判断以 `resources/Template.xlsx`、`resources/生产报告单.pdf`、`resources/2026年激光统计表.xls` 为证据来源。

完成标准：

- 根目录 `PLAN.md` 已记录阶段、任务和验证命令。
- README 已说明 backend-first、MSSQL 2016、OpenAPI 和 `.env` 交付方式。
- API 字段、状态 key、认证方式和数据库目标不再处于“未决定”状态。

## 阶段 2：数据模型

目标是设计可落库、可追溯、可统计的数据结构。

建议先设计这些实体：

- 用户与 token 撤销记录。
- 材质、厚度和物料规格。
- 整料库存。
- 余料库存。
- 废料记录。
- 每日备料单。
- 备料明细，覆盖 `Template.xlsx` 的板材名称、图纸路径、宽、长、材质、厚度、数量。
- 生产报告 PDF 文件记录。
- 生产报告解析指标。
- 余料确认记录。
- 废料确认记录。
- 日结记录。

完成标准：

- 核心实体有主键、业务编号、状态、创建时间和更新时间。
- 整料、余料、废料记录必须能追溯到日期、来源、材质、厚度、宽长、数量和状态。
- 日结后原则上不直接修改原始记录；需要调整时保留调整记录或重开确认流程。

## 阶段 3：技术栈选择

当前后端技术栈已确定：

- 后端：FastAPI。
- ORM / Migration：SQLAlchemy 2.x + Alembic。
- 数据库：Microsoft SQL Server 2016。
- DB driver：`mssql+pyodbc`。
- 权限：MVP 使用 JWT + 简单角色权限。
- 开发环境：`docker-compose` 可用于本地数据库和冒烟验证。
- 交付：backend 可执行程序或可执行目录，通过 `.env` 连接现场数据库。

暂不决定：

- Qt 或 Vue 前端的具体实现顺序。
- 是否引入 Worker / Celery，等 PDF 批处理或耗时任务规模明确后再定。
- 最终可执行打包工具。

完成标准：

- 明确开发环境和运行方式。
- 明确数据库和迁移方案。
- 明确第一版先做 backend API，OpenAPI 供 Qt / Vue 使用。

## 阶段 4：工程骨架

目标是生成可运行的最小 backend 工程，而不是一次性实现全部业务。

当前顺序：

1. 后端项目骨架：已完成。
2. 数据库迁移机制：已完成。
3. `.env.example` 和 MSSQL 2016 连接配置：已完成。
4. 健康检查、统一响应、OpenAPI 契约和认证：已完成。
5. 物料 / 余料库存 API 第一版：已完成。
6. 下一步：每日备料单、库存来源选择和 `Template.xlsx` 导出。

完成标准：

- 本地能启动后端。
- 有最小测试或检查命令。
- README 记录运行方式。
- 物料 / 余料库存 API 可以演示；第一条备料 / 导出闭环是下一步。

## 阶段 5：MVP 闭环

目标是完成第一条可用业务链路：

```text
今日排产任务
  ↓
整料 / 余料库存
  ↓
每日备料单
  ↓
Template.xlsx 导出
  ↓
生产报告 PDF 导入
  ↓
余料 / 废料确认
  ↓
日结
  ↓
月结
```

完成标准：

- 可以维护整料和余料库存。
- 可以生成设备可导入的 `Template.xlsx`。
- 可以导入生产报告 PDF 并保存解析状态。
- 可以登记余料和废料确认结果。
- 可以生成日结数据。
- 可以按月汇总基础指标。
- 可以导出绩效计算所需的基础数据。

## 暂缓事项

第一版暂缓：

- 自动化设备控制。
- 复杂绩效规则引擎。
- 完整采购、销售、财务模块。
- 高级排产。
- 仓库波次和路径优化。
- 多工厂、多组织复杂权限。

这些能力可以在 MVP 跑通后再拆成后续阶段。
